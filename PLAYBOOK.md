# 📋 Playbook — ทำตามนี้ทีละขั้นในห้องสอบ (กันงง)

ลำดับนี้เป๊ะ ๆ ทำตามไม่ต้องคิดเยอะ เน้น "มีคะแนนก่อน แล้วค่อยดัน"

## ก่อนเริ่ม (5 นาทีแรก)
1. เปิดหน้า Kaggle ของโจทย์ กด `Join Competition` + ยอมรับ rules (ไม่งั้นโหลด/ส่งไม่ได้)
2. อ่าน tab `Data` (มีไฟล์อะไร) + `Evaluation` (วัดด้วย metric อะไร) + ดู `จำนวนส่งต่อวัน`
3. ถ้างานเป็น deep learning (รูป/เสียง/ข้อความยาว) เปิด GPU: `Runtime > Change runtime type > T4 GPU`

## เลือกเครื่องมือ (1 นาที)
4. ขี้เกียจ/รีบ -> เปิด `00_AUTO_SOLVER.ipynb` (รูป/ข้อความ/ตาราง ทำให้อัตโนมัติ)
   อยากคุมเอง/งานเฉพาะ -> เปิด `00_MASTER_ROUTER.ipynb` พิมพ์โจทย์ -> มันบอกโน้ตบุ๊ก

## ลงมือ (เรียงตามนี้)
5. รันเซลล์ติดตั้ง + เซลล์ใส่ข้อมูล (Kaggle API หรืออัปโหลด zip ตั้ง `DATA_DIR`)
6. รันเซลล์ `🔎 ดูโจทย์` -> รู้ว่าต้องเติมคอลัมน์อะไร + metric อะไร + ส่งเป็น ป้าย/prob/ตัวเลข
7. รันเซลล์ `👀 EDA` -> ดูคลาสสมดุลไหม มีค่าว่างไหม (AUTO_SOLVER มีให้)
8. ส่ง `baseline ก่อนเลย`! (AUTO_SOLVER สร้าง `submission_baseline.csv` ให้) -> ล็อกว่าฟอร์แมตถูก + ได้คะแนนพื้น
9. รันแบบ `เร็ว` ก่อน: ตั้ง `time_limit=120` (หรือ `EPOCHS` น้อย) ให้รันผ่านจนได้ submission 1 ไฟล์
10. ดู `คะแนน validation ในเครื่อง` -> ถ้าโอเค ค่อยรันจริง `time_limit` มากขึ้น (เหลือเวลาก็เพิ่ม)
11. ส่งไฟล์จริง -> เช็คคะแนนบนกระดาน

## ดันคะแนน (ถ้ามีเวลา)
12. ทำอีก 1-2 วิธี (วิธีที่ 2/3 ในโน้ตบุ๊ก หรือเปลี่ยนโมเดล) ให้ได้หลายไฟล์ submission
13. รันเซลล์ `blend` -> รวมหลายไฟล์เป็น `submission_blend.csv` (มักได้เพิ่มฟรี ๆ)
14. ดู `ADVANCED_SOTA.md` หัวข้อของคุณ -> เปลี่ยนโมเดลเป็นตัวแรง ๆ ตามที่บอก

## ตอนใกล้หมดเวลา
15. เลือกไฟล์ส่งสุดท้ายจาก `คะแนน CV ในเครื่อง` ไม่ใช่ public LB อย่างเดียว (LB หลอกได้)
16. เก็บสิทธิ์ส่งไว้ 1 ครั้ง เผื่อแก้ฟอร์แมตนาทีสุดท้าย
17. ยืนยันว่า submission สุดท้าย "เลือกแล้ว" บนหน้า Kaggle (บางที่ต้องเลือก final)

## กฎทอง (ท่องไว้)
- มีคะแนนก่อนเสมอ (baseline) แล้วค่อยทำให้ดีขึ้น
- เริ่มจากของง่าย (AUTO/AutoGluon) แล้วค่อยขั้นสูง
- เชื่อ CV ในเครื่อง มากกว่า public LB
- เจอ error -> เปิด `TROUBLESHOOTING.md`
- ไม่รู้ว่าโจทย์แปลกนี้ทำไง -> เปิด `PROBLEM_TYPES.md`
